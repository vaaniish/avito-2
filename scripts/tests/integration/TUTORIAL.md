# Руководство по integration-тестам

## Что здесь есть

- integration-тесты проверяют реальные связки `модуль + БД + инфраструктура`
- типичный пример — сценарии policy acceptance, delivery fallback, complaint cascade, listing moderation и payout flow

## Когда запускать

- После изменений в БД-схеме
- После изменений в роутинге и сервисной логике
- Перед e2e, чтобы быстро убедиться, что слой `модуль + БД` не сломан

## Как запускать

```bash
npm run db:migrate:deploy
npm run db:seed
npm run test:integration
```

## Как анализировать падения

- Проверять сиды и миграции
- Проверять, что тест идёт в локальную и безопасную БД
- Проверять ограничения, индексы и бизнес-инварианты, связанные с кейсом
- Смотреть конкретный `AssertionError` и цепочку вызовов

## Практика

- Integration-тесты должны проверять только важные связки
- Для каждой критичной БД-логики желательно иметь хотя бы `happy path + rejection path`
