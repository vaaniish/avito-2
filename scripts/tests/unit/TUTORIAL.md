# Unit Tests Tutorial

## Что здесь сейчас есть
- `order-status-fsm.unit.test.ts`: инварианты переходов статусов заказа.
- `anti-circumvention.unit.test.ts`: детектор сигналов увода сделки (телефон, email, ссылки, фразы и т.д.).
- `session-token.unit.test.ts`: подпись/проверка сессий, истечение, базовые правила production-конфига.

## Когда запускать
- После любых изменений в доменной логике.
- Перед integration/e2e, чтобы быстро поймать базовые регрессии.

## Как запускать
```bash
npm run test:unit
```

## Как анализировать
- Unit не открываются “через UI приложения”, это проверка функций.
- Основной сигнал:
  - `PASS` по всем тестам
  - отсутствие flaky поведения
- При `FAIL`:
  - смотри имя теста
  - смотри `AssertionError` и expected/actual
  - смотри файл и строку в стектрейсе

## Практика в Agile
- Unit должны быть маленькими, быстрыми и стабильными.
- На каждую новую бизнес-ветку добавляй 1-2 точечных unit-теста.
- На каждый найденный баг сначала тест, затем фикс.
