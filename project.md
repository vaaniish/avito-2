# Анализ фронтенд кодовой базы: My Product Gamification

## 📁 Структура проекта

Код фронтенда организован по принципу разделения на слои (Layer-based) с элементами разделения по функциональным ролям (Admin/Partner).

```text
frontend/
├── src/
│   ├── components/       # UI компоненты
│   │   ├── admin/        # Компоненты панели администратора
│   │   ├── pages/        # Основные страницы приложения
│   │   ├── partner/      # Компоненты личного кабинета партнера
│   │   └── ui/           # Переиспользуемые атомарные UI-компоненты
│   ├── guidelines/       # Документация и руководства по стилю
│   ├── lib/              # Утилиты, API-клиент и общая логика
│   ├── styles/           # Глобальные стили и конфигурации CSS
│   ├── App.tsx           # Корневой компонент (Routing & Global State)
│   ├── main.tsx          # Точка входа
│   └── types.ts          # Общие TypeScript интерфейсы
├── index.html            # HTML-шаблон
├── tsconfig.json         # Конфигурация TypeScript
└── vite.config.ts        # Конфигурация сборщика Vite
```

**Описание директорий:**
- `components/ui`: Содержит низкоуровневые компоненты (кнопки, модалки), часто с использованием Radix UI.
- `lib/api.ts`: Централизованный слой для взаимодействия с бэкендом через `fetch`.
- `App.tsx`: Выполняет роль роутера и хранилища основного состояния приложения.

## 🏗 Архитектура

### Маршрутизация (Routing)
В проекте используется **State-based routing** вместо стандартного `react-router`. Текущая страница определяется состоянием `currentView` в `App.tsx`.
```typescript
type AppView = "home" | "cart" | "checkout" | "product" | "adminPanel" | ...;
const [currentView, setCurrentView] = useState<AppView>("home");
```

### Управление состоянием
Используется **Local State Management** на уровне корневого компонента `App.tsx`. Глобальное состояние (корзина, данные пользователя, фильтры) передается через пропсы вниз по дереву компонентов. Это подходит для средних проектов, но может привести к "prop drilling".

### Слой данных (API Layer)
Работа с сетью абстрагирована в `lib/api.ts`. Используется типизированная обертка над `fetch` с поддержкой авторизации через `localStorage` и кастомные заголовки (`x-user-id`).

## 🎨 UI/UX и стилизация
- **Tailwind CSS**: Основной инструмент стилизации. Классы пишутся непосредственно в JSX.
- **Интерактивность**: Используется `motion` (Framer Motion) для плавных переходов и эффектов (например, в `GlowButton`).
- **Компонентный подход**: Сложные компоненты собираются из мелких `ui`-примитивов.
- **Адаптивность**: Реализована через стандартные модификаторы Tailwind (`md:`, `lg:`).

## ✅ Качество кода
- **TypeScript**: Активное использование интерфейсов для моделей данных (`Product`, `User`). Это обеспечивает высокую типобезопасность.
- **Линтинг**: В проекте настроен `eslint` (упоминается в зависимостях среды).
- **Тестирование**: Автоматизированные тесты (Unit/E2E) на данный момент **отсутствуют**.
- **Соглашения**: Используется PascalCase для компонентов и camelCase для функций/переменных. Код чистый и хорошо структурирован.

### 2. `ProductCard` (Бизнес-компонент)
Универсальный компонент для отображения товара в двух режимах: `grid` и `list`.
- **Props**: `product`, `viewMode`, `displayMode`.
- **Логика**: Обработка добавления в корзину, wishlist и расчет скидок.

## 📋 Выводы и рекомендации

### Сильные стороны:
1. **Чистая структура**: Понятное разделение компонентов по ролям и ответственности.
2. **Типизация**: Хорошо проработанные типы данных снижают вероятность ошибок.
3. **UX/UI**: Высокое внимание к деталям и микро-взаимодействиям (анимации, свечения).

### Области для улучшения:
1. **Маршрутизация**: Рекомендуется внедрить `react-router-dom` для поддержки истории браузера и прямой навигации по URL.
2. **State Management**: При росте проекта стоит рассмотреть `Zustand` или `React Context`, чтобы избежать "prop drilling" в `App.tsx`.
3. **Тестирование**: Необходимо внедрить `Vitest` для unit-тестов логики корзины и фильтрации.
4. **API**: Использование `TanStack Query (React Query)` значительно упростило бы обработку состояний загрузки и кэширование данных.
